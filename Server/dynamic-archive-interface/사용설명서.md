<h2 id="2-System-Architecture">💡 2.작동 방식</h2>

라이브러리 없이 순수 자바스크립트(Vanilla JS)와 최신 CSS 기능을 결합한 구조입니다.

- Metadata Mapping: 모든 UI 요소는 projects 객체 배열을 통해 생성되므로 데이터와 뷰가 분리되어 유지보수가 용이합니다.

- Recursive Image Loader: 리소스 로딩 실패 시 jpg → png → gif 순으로 확장자를 자동 탐색하며 브라우저 에러를 방지합니다.

- Interactive UX: 모달 창은 방향키와 ESC 키를 활용한 키보드 네비게이션을 지원해 조작을 직관적으로 만들었습니다.

<h2 id="3-Upload-Expansion">💡 3.업로드 및 확장</h2>

1. images/ 폴더 내부에 YYMMDD 형식의 새 폴더를 만들고 이미지들을 저장합니다.

> [!IMPORTANT]
> 수의 순서대로 파일을 읽어옵니다. 예를 들어 1.jpg는 썸네일이 되는 동시에 시작점이되어 1.jpg -> 2.png순으로 내용을 볼 수 있게됩니다.
  
3. projects 배열에 새 객체를 추가하고 folder, size, count 등의 메타데이터를 설정합니다.
- 1. ```const projects = [``` 을 찾습니다.
   
> [!TIP]
> 메모장 프로그램으로 html파일을 열었을 시 ctrl + f 키를 통해 빠르게 위치를 찾을 수 있습니다.

- 2.양식에 맞게 내용을 입력합니다.

```bash
const projects = [
        { folder: "000000", title: "test", size: "large", count: 1, category: "brand" },
    ];
```

folder: "000000" :: 연도/달/요일의 입력대로 작성한 폴더의 이름을 그대로 작성합니다. 이 경우 00年 00月 00日로 인식하게됩니다.

title: "test" :: 내용을 클릭해 확인하지않아도 확인할 수 있는 기본적인 설명입니다. 이 경우 test가 설명으로 출력됩니다

size: "large" :: 표시되는 크기를 지정합니다. 크기의 경우 large, normal, wide, medium으로 지정해줄 수 있습니다.

count: 1 :: page의 수만큼 입력하면 됩니다. 이 경우 이미지가 1개임을 명시합니다.

category: "brand" :: 카테고리 정렬에서 사용되는 태그입니다. 시안에선 쓰이지않았습니다. brand, ad, promotion등 목적에 맞는 태그를 넣어 사용할 수 있습니다.
